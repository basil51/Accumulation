# Beta Testing Guide

Welcome to the Accumulation Platform Beta! This guide will help you get started and provide feedback.

---

## What is Beta Testing?

Beta testing allows us to gather real-world feedback from actual users before the public launch. Your participation helps us:

- Identify bugs and issues
- Improve user experience
- Validate features and workflows
- Optimize detection accuracy
- Refine the platform for production

---

## Getting Started

### 1. Create Your Account

1. Navigate to the signup page
2. Enter your email and create a password
3. Verify your email (if email verification is enabled)
4. Log in to your account

### 2. Explore the Dashboard

- **Dashboard**: View recent accumulation and market signals
- **Signals Page**: Browse all signals with filtering options
- **Watchlist**: Add coins to track
- **Alerts**: View your alerts and notifications
- **Settings**: Configure your preferences

### 3. Set Up Your Watchlist

1. Go to **Watchlist** page
2. Select a chain (Ethereum, Polygon, Arbitrum, etc.)
3. Search for a coin by symbol
4. Add it to your watchlist
5. Configure thresholds and notifications

### 4. Configure Alert Preferences

1. Go to **Settings** page
2. Set your minimum signal score (default: 65)
3. Configure cooldown period (default: 30 minutes)
4. Enable/disable email and Telegram notifications

## How to Provide Feedback

### Feedback Types

1. **Bug Reports**: Something isn't working as expected
2. **Feature Requests**: Ideas for new features or improvements
3. **UI/UX Feedback**: Interface and user experience suggestions
4. **Signal Quality**: Feedback on detection accuracy
5. **Performance Issues**: Slow loading or responsiveness problems

### Feedback Form

Use the feedback form in the Settings page or contact us directly:

- **Email**: [Your support email]
- **Feedback Form**: Settings → Feedback

### What to Include

When reporting issues or providing feedback:

1. **Description**: Clear description of the issue or suggestion
2. **Steps to Reproduce**: For bugs, include steps to reproduce
3. **Expected vs Actual**: What you expected vs what happened
4. **Screenshots**: If applicable, include screenshots
5. **Browser/Device**: Browser version and device type
6. **Account Info**: Your subscription level (if relevant)

## Testing Scenarios

### Scenario 1: Basic Signal Detection

1. Add a popular coin (e.g., ETH, BTC) to your watchlist
2. Wait for signals to appear
3. Review signal details and scores
4. Check if alerts are received correctly

### Scenario 2: Multi-Chain Testing

1. Add coins from different chains (Ethereum, Polygon, Arbitrum)
2. Verify signals appear for all chains
3. Test chain-specific filtering

### Scenario 3: Alert Configuration

1. Set minimum signal score to 70
2. Configure cooldown to 60 minutes
3. Verify only high-quality signals trigger alerts
4. Test notification delivery

### Scenario 4: False Positive Reporting

1. Review signals in the dashboard
2. Identify any false positives
3. Report false positives through feedback form
4. Note which signals were incorrectly flagged

## Best Practices

### For Accurate Testing

1. **Use Real Data**: Test with actual coins and real signals
2. **Test Different Scenarios**: Try various coins, chains, and configurations
3. **Document Issues**: Take notes on bugs and issues you encounter
4. **Provide Detailed Feedback**: Include as much context as possible

### For Signal Quality

1. **Review Signal Details**: Check evidence and score breakdown
2. **Compare with Market Data**: Verify signals against actual market activity
3. **Report False Positives**: Help improve detection accuracy
4. **Note Missing Signals**: Report if you notice missed accumulation events
